# Agora: Revised Execution Plan

## 1. IDENTITY & SECURITY (The "Non-Brutable" Identity)

We are moving from "Username+Password" to a **"Passphrase Identity"** model to prevent brute-force attacks and identity confusion.

*   **Derivation**: Switch to **Argon2id** (via `@noble/hashes`). It is more resistant to GPU cracking than PBKDF2.
*   **Entropy Requirement**: Minimum 12 characters. Use an on-screen "Strength Meter."
*   **Identity Clarity**: Usernames are display names only. The **npub** is the unique ID. We will display a `short-npub` (e.g., `npub1...abcd`) next to every username to prevent impersonation.
*   **Worker-Based Derivation**: Move Argon2 to a **Web Worker**. This prevents the UI from freezing on mobile devices during the 2-5 second derivation process.
*   **NIP-05 Support**: Add a free "Verification" field where users can paste a NIP-05 identifier if they have one.

## 2. SYBIL RESISTANCE (Anti-Spam)

To prevent an attacker from creating 1,000 accounts to manipulate the algorithm for free:

*   **NIP-13 Proof of Work (PoW)**: The client will mine a small difficulty (e.g., 16-20 bits) for every post. This takes ~1-3 seconds on a phone but makes mass-botting expensive.
*   **Weighted Scoring**: The Collaborative Algorithm now weights votes based on the **account's age** (first seen timestamp) and **PoW difficulty** of the vote event. A vote from a 1-year-old account counts more than a vote from a 1-minute-old account.
*   **Audit Log**: All moderation actions and community weight changes are published as Nostr events.

## 3. RELIABLE MEDIA (The Redundant Layer)

To fix the fragility of free hosting:

*   **Multi-Publish**: Files are uploaded to **two** NIP-96 servers simultaneously. The Nostr event stores both URLs.
*   **Capability Discovery**: The client fetches `/.well-known/nostr/nip96.json` from servers to determine their specific upload limits and auth requirements before attempting upload.
*   **Metadata-Only Stripping**: Instead of re-encoding images (lossy), we use a byte-stripping approach to remove EXIF tags while preserving the original image data.

```typescript
// src/lib/crypto/worker.ts -> Argon2id in a background thread
import { argon2id } from '@noble/hashes/argon2';

export async function derive(passphrase: string, salt: string) {
  // Argon2id parameters tuned for mobile (Memory: 64MB, Iterations: 3)
  return argon2id({
    password: passphrase,
    salt: salt,
    t: 3, 
    m: 65536, 
    p: 1, 
    dkLen: 32
  });
}
```

### 5.2 Sybil-Resistant Scoring
```typescript
// src/lib/algorithm/scoring.ts
export function calculateWeight(event: NostrEvent) {
  const pow = getPowDifficulty(event.id); // NIP-13
  const accountAge = Date.now() - (event.created_at * 1000);
  
  // Weights: Older accounts and higher PoW = higher signal
  const ageFactor = Math.log1p(accountAge / 86400000); 
  return pow * ageFactor;
}
```

PLAN BEFORE UPDATES :
**The core problem to solve:**

The previous version assumed users are privacy-maximalists who will configure Tor, manage relay connections, and understand cryptographic concepts. That's a tiny minority. Real users want it to just work — and we still want to preserve the decentralized, privacy-respecting architecture underneath.

**What are we actually solving for?**
- Transport privacy without requiring Tor
- File storage without requiring users to set up pinning service API keys
- Identity without requiring users to understand keypairs
- The whole thing working on mobile browsers, not just desktop with extensions

**Let me think through each concern and find always-free, no-configuration alternatives:**

---

**TRANSPORT PRIVACY (instead of Tor)**

Options:
1. **Tor** — requires configuration, breaks on mobile, slow
2. **VPN** — costs money, requires trust in VPN provider
3. **Just use WSS (TLS WebSockets)** — this IS the baseline. Relay operator sees your IP, ISP sees you connecting to relay domain but not content
4. **Cloudflare for relay connections** — many relays already sit behind Cloudflare, which hides the origin server's IP
5. **Always-free privacy proxies** — don't exist reliably
6. **Privacy-by-architecture instead of privacy-by-transport** — design so that even if the relay sees your IP, the damage is limited (minimize what you POST, not just how you connect)

**Decision**: Don't try to hide transport. Instead:
- Use WSS (TLS) everywhere — content is encrypted in transit by default
- Let users know what the relay sees (their IP and public events — that's it for public posts)
- For truly sensitive use cases, recommend Tor Browser as an option but don't require it
- Focus privacy engineering on: key separation, what data we collect, what we store

This is honest and achievable. Most social platforms have the same transport privacy model (TLS). The difference is we don't have a central server that aggregates everything.

---

**FILE STORAGE (instead of requiring Pinata/web3.storage API keys)**

Options:
1. **Nostr.build** — free Nostr media hosting, no account needed, just upload
2. **void.cat** — free, anonymous file hosting used by Nostr community
3. **Blossom protocol** — new Nostr media spec (NIP-96 / BUD-01), servers host media for Nostr users
4. **NIP-96** — HTTP File Storage Integration spec. Multiple free servers implement it. No API key needed for some.
5. **Imgur/Catbox** — free image hosting, but centralized, not great for privacy
6. **Keep Helia/IPFS locally** — browser node pins it locally, and web3.storage free tier works with a simple email signup (no credit card). 5GB free.
7. **Mix**: Use NIP-96 servers as primary (zero config, just works), IPFS as optional enhancement

**Decision**: Use **NIP-96** as the primary media layer. NIP-96 is a Nostr standard for HTTP file storage. Multiple servers implement it:
- `https://nostr.build` — free, NIP-96 compliant
- `https://void.cat` — free, NIP-96 compliant  
- `https://nostrcheck.me` — free, NIP-96 compliant
- `https://files.sovbit.host` — free

No API keys. No account. Just POST the file, get back a URL. The user can configure which server they prefer but the default just works.

IPFS remains as an **optional** enhancement layer for users who want it — but it's not required.

---

**IDENTITY (instead of requiring users to understand keypairs)**

Options:
1. **NIP-07 browser extensions** — nos2x, Alby — but requires installation
2. **NIP-46 remote signer** — nsecBunker — requires setup
3. **Generate key in browser, store encrypted** — works but "write down your 24 words" is scary for normal users
4. **Magic link via Nostr** — doesn't make sense
5. **Social login → derive Nostr key** — defeats privacy
6. **Email + derive key server-side** — requires a server
7. **Simple username + password → derive key deterministically** — user just needs to remember username+password, key is derived via Argon2/PBKDF2. Same credentials = same key everywhere. No seed phrase needed.
8. **Optional: NIP-07 detected and preferred** if available

**Decision**: 
- Default: **username + password → deterministically derive Nostr keypair via Argon2id**
- User just picks a username and password — they'll have the same key on every device
- No seed phrases, no backups required (password IS the backup)
- If NIP-07 extension detected, offer to use it instead
- This is similar to how Amber (Android Nostr signer) and some other apps work
- Warn: "If you forget your password, there is no recovery. We don't store it."

---

**RELAY SELECTION (instead of requiring users to configure)**

Decision: Opinionated defaults. Pre-select a curated set of reliable, free public relays. User never needs to touch this. Advanced settings available for power users.

---

**IPFS / FILE STORAGE (revised)**

Three tiers, all free, in order of simplicity:
1. **NIP-96 servers** (default, zero config) — just works, multiple free servers, Nostr-native
2. **IPFS via Helia** (optional enhancement) — for users who want true decentralization
3. **web3.storage** (optional, email signup required) — for guaranteed persistence

For the default experience: NIP-96. Files go to nostr.build or similar. URL is in the Nostr event. Works on mobile. No configuration.

**ENCRYPTION**

Instead of requiring users to set up group keys and understand Double Ratchet:
- Public posts: plaintext (that's the point of a forum)
- Private DMs: **NIP-44** (built into nostr-tools v2, just works)
- Private communities: Start simple — invite-only communities where membership is gated by community approval. Full E2E encryption is Phase 2.

## 5. IDENTITY SYSTEM (The Key Decision)

### 5.1 How It Works for Users

```
User sees:        "Choose a username and password"
Under the hood:   PBKDF2(password, salt=username, iterations=250000)
                  → 32 bytes → secp256k1 private key → Nostr keypair
Result:           Same username+password = same keypair, always, everywhere
                  No seed phrase. No backup file. Password IS the key.
```

### 5.2 Why This Approach

| Concern | Solution |
|---|---|
| Works on mobile | ✅ No extension required |
| No seed phrase | ✅ Password is the recovery mechanism |
| Cross-device | ✅ Same password = same key on any device |
| Power user escape hatch | ✅ Can import raw nsec or use NIP-07 |
| Honest security | ⚠️ Key strength = password strength. Document this clearly. |

### 5.3 Key Derivation

```typescript
// src/lib/crypto/derive.ts
import { pbkdf2 } from '@noble/hashes/pbkdf2'
import { sha256 } from '@noble/hashes/sha256'
import { secp256k1 } from '@noble/curves/secp256k1'
import { bytesToHex } from '@noble/hashes/utils'
import { getPublicKey, nip19 } from 'nostr-tools'

export interface DerivedKeypair {
  secretKey: Uint8Array
  publicKey: string    // hex
  npub: string
  nsec: string
}

export function deriveKeypairFromPassword(
  username: string,
  password: string
): DerivedKeypair {
  // Username is the salt — ensures same password → different key per username
  // This also means you cannot use the same password for different usernames
  // and get the same key (intentional — each username is a distinct identity)
  const salt = new TextEncoder().encode(`agora:v1:${username.toLowerCase().trim()}`)
  const pass = new TextEncoder().encode(password)

  // 250,000 iterations — roughly 500ms on modern hardware
  // Expensive enough to resist brute force, fast enough to not frustrate users
  const secretKey = pbkdf2(sha256, pass, salt, { c: 250_000, dkLen: 32 })

  // Ensure the key is valid for secp256k1 (almost certainly is, but check)
  // If not, increment a counter and retry (deterministic)
  let finalKey = secretKey
  let attempt = 0
  while (!secp256k1.utils.isValidPrivateKey(finalKey)) {
    const saltWithCounter = new TextEncoder().encode(`agora:v1:${username}:${++attempt}`)
    finalKey = pbkdf2(sha256, pass, saltWithCounter, { c: 250_000, dkLen: 32 })
  }

  const publicKey = bytesToHex(secp256k1.getPublicKey(finalKey, true).slice(1))

  return {
    secretKey: finalKey,
    publicKey,
    npub: nip19.npubEncode(publicKey),
    nsec: nip19.nsecEncode(finalKey),
  }
}

// For power users: import existing nsec directly
export function keypairFromNsec(nsec: string): DerivedKeypair {
  const { data: secretKey } = nip19.decode(nsec)
  if (!(secretKey instanceof Uint8Array)) throw new Error('Invalid nsec')
  const publicKey = bytesToHex(secp256k1.getPublicKey(secretKey, true).slice(1))
  return {
    secretKey,
    publicKey,
    npub: nip19.npubEncode(publicKey),
    nsec,
  }
}
```

### 5.4 Personas (Multiple Identities)

For users who want multiple identities, we derive additional keypairs deterministically:

```typescript
// Additional personas: derived from primary key + persona index
// Username "alice" password "pw" → primary key
// Username "alice:1" password "pw" → pseudonym 1 (same password, different username prefix)
// Username "alice:anon" password "pw" → anonymous persona

// For throwaway/anonymous posting: generate ephemeral key in browser
// Not derived from password — truly one-time, discarded after session
export function generateEphemeralKeypair(): DerivedKeypair {
  const secretKey = secp256k1.utils.randomPrivateKey()
  const publicKey = bytesToHex(secp256k1.getPublicKey(secretKey, true).slice(1))
  return {
    secretKey,
    publicKey,
    npub: nip19.npubEncode(publicKey),
    nsec: nip19.nsecEncode(secretKey),
  }
}
```

### 6.5 NIP-10 Thread Tree

```typescript
// src/lib/nostr/threading.ts
import type { Event as NostrEvent } from 'nostr-tools'

export interface TreeNode {
  event: NostrEvent
  children: TreeNode[]
  depth: number
}

export function buildTree(root: NostrEvent, replies: NostrEvent[]): TreeNode[] {
  const map = new Map<string, TreeNode>()

  for (const e of replies) {
    map.set(e.id, { event: e, children: [], depth: 0 })
  }

  const roots: TreeNode[] = []

  for (const e of replies) {
    const node = map.get(e.id)!
    // NIP-10: find reply marker, fallback to last e tag
    const eTags = e.tags.filter(t => t[0] === 'e')
    const replyTag = eTags.find(t => t[3] === 'reply') ?? eTags.find(t => t[3] !== 'root')
    const parentId = replyTag?.[1]

    if (parentId === root.id || !parentId) {
      roots.push(node)
    } else if (map.has(parentId)) {
      const parent = map.get(parentId)!
      node.depth = parent.depth + 1
      parent.children.push(node)
    } else {
      roots.push(node)  // orphan → show at root
    }
  }

  return roots
}
```

## 7. MEDIA LAYER (NIP-96 PRIMARY)

### 7.1 What NIP-96 Is

NIP-96 defines a standard HTTP API for file hosting servers that Nostr clients can use. Multiple free servers implement it. Upload a file, get back a URL. No account. No API key. Works on mobile.

```
POST https://nostr.build/api/v2/nip96/upload
  Body: multipart/form-data with the file
  Auth: Nostr event signature (proves ownership of pubkey)
Response: { url, sha256, ... }
```

### 7.2 NIP-96 Client

```typescript
// src/lib/media/nip96.ts

// User never needs to configure these — we try them in order

export interface UploadResult {
  url: string
  sha256: string
  mimeType: string
  originalSize: number
  dimensions?: { width: number; height: number }
  serverName: string
}

// ─── Main Upload Function ─────────────────────────────────────────

export async function uploadMedia(
  file: File,
  onProgress?: (pct: number) => void
): Promise<UploadResult> {
  const persona = get(activePersona)
  if (!persona) throw new Error('Must be logged in to upload')

  // Step 1: Strip EXIF from images
  const processedFile = file.type.startsWith('image/')
    ? await stripExif(file)
    : file

  // Step 2: Try each server in order
  for (const server of NIP96_SERVERS) {
    try {
      const result = await uploadToServer(processedFile, server, persona.keypair.secretKey, onProgress)
      return result
    } catch (err) {
      console.warn(`[NIP-96] ${server.name} failed:`, err)
      // Try next server
    }
  }

  throw new Error('All media servers failed. Check your connection.')
}

async function uploadToServer(
  file: File,
  server: Nip96Server,
  secretKey: Uint8Array,
  onProgress?: (pct: number) => void
): Promise<UploadResult> {

  // NIP-96 auth: sign a special authorization event
  const authEvent = finalizeEvent({
    kind: 27235,  // NIP-98 HTTP Auth
    created_at: Math.floor(Date.now() / 1000),
    tags: [
      ['u', server.apiUrl],
      ['method', 'POST'],
    ],
    content: '',
  }, secretKey)

  const authHeader = `Nostr ${btoa(JSON.stringify(authEvent))}`

  const form = new FormData()
  form.append('file[]', file)

  // Use XMLHttpRequest for progress tracking
  const result = await new Promise<UploadResult>((resolve, reject) => {
    const xhr = new XMLHttpRequest()
    xhr.open('POST', server.apiUrl)
    xhr.setRequestHeader('Authorization', authHeader)

    xhr.upload.onprogress = (e) => {
      if (e.lengthComputable) onProgress?.(Math.round(e.loaded / e.total * 100))
    }

    xhr.onload = () => {
      if (xhr.status >= 200 && xhr.status < 300) {
        try {
          const data = JSON.parse(xhr.responseText)
          // nostr.build response format
          const fileData = data.nip94_event?.tags ?? []
          const url = fileData.find((t: string[]) => t[0] === 'url')?.[1]
            ?? data.url ?? data.data?.links?.web
          const sha256 = fileData.find((t: string[]) => t[0] === 'x')?.[1]
          if (!url) throw new Error('No URL in response')
          resolve({
            url,
            sha256: sha256 ?? '',
            mimeType: file.type,
            originalSize: file.size,
            serverName: server.name,
          })
        } catch (e) { reject(e) }
      } else {
        reject(new Error(`HTTP ${xhr.status}: ${xhr.statusText}`))
      }
    }
    xhr.onerror = () => reject(new Error('Network error'))
    xhr.send(form)
  })

  return result
}

// ─── EXIF Stripping ───────────────────────────────────────────────
// Re-encode via OffscreenCanvas to strip all metadata

async function stripExif(file: File): Promise<File> {
  const bitmap = await createImageBitmap(file)
  const canvas = new OffscreenCanvas(bitmap.width, bitmap.height)
  const ctx = canvas.getContext('2d')!
  ctx.drawImage(bitmap, 0, 0)
  const blob = await canvas.convertToBlob({ type: 'image/webp', quality: 0.88 })
  return new File([blob], file.name.replace(/\.[^.]+$/, '.webp'), { type: 'image/webp' })
}
```

## 8. PRIVACY LAYER

### 8.1 Honest Privacy Model

```
WHAT WE PROTECT:
  ✅ Local data (AES-GCM encrypted with password-derived key)
  ✅ Private messages (NIP-44 + NIP-59 gift wrap)
  ✅ Image metadata (EXIF stripped before upload)
  ✅ Multiple personas are unlinkable (different keypairs)
  ✅ Password never stored — re-derived on login
  ✅ No tracking, no analytics, no fingerprinting by us

WHAT WE DO NOT PROTECT (AND WHY):
  ❌ Your IP from relay operators
      → We use TLS WebSockets. Relay sees your IP.
      → This is the same as any website. Accept it or use Tor.
      → We recommend Tor Browser for sensitive use cases.

  ❌ Your IP from NIP-96 media servers
      → File uploads go to nostr.build etc. over HTTPS.
      → Same situation as uploading to Imgur.

  ❌ Public posts are public
      → Forum posts on public relays are readable by anyone.
      → This is by design. It's a forum.

  ❌ Stylometric fingerprinting
      → Your writing style can identify you across personas.
      → No technical solution exists.

  ❌ Browser fingerprinting across sessions
      → Without Tor Browser, browser fingerprints are linkable.
      → Use Tor Browser or Firefox with privacy extensions
        if this matters to you.

WE RECOMMEND FOR SENSITIVE USE:
  → Tor Browser (completely separate browser fingerprint,
    routes all traffic through Tor including WebSockets)
  → We support Tor. We don't require it.
```

### 8.2 Encrypted Local Storage

```typescript
// src/lib/crypto/storage.ts
// All Agora data in localStorage/IndexedDB is encrypted at rest.
// Encryption key derived from user's login password.
// If someone gets the device, they get ciphertext.

const SALT_KEY = 'agora:storage:salt'

function getSalt(): Uint8Array {
  const stored = localStorage.getItem(SALT_KEY)
  if (stored) return new Uint8Array(JSON.parse(stored))
  const salt = crypto.getRandomValues(new Uint8Array(32))
  localStorage.setItem(SALT_KEY, JSON.stringify(Array.from(salt)))
  return salt
}

export class EncryptedStore {
  private key: CryptoKey | null = null

  async init(password: string): Promise<void> {
    const salt = getSalt()
    const base = await crypto.subtle.importKey(
      'raw', new TextEncoder().encode(password), 'PBKDF2', false, ['deriveKey']
    )
    this.key = await crypto.subtle.deriveKey(
      { name: 'PBKDF2', salt, iterations: 250_000, hash: 'SHA-256' },
      base, { name: 'AES-GCM', length: 256 }, false, ['encrypt', 'decrypt']
    )
  }

  async set(k: string, v: unknown): Promise<void> {
    if (!this.key) throw new Error('Store not initialized')
    const iv = crypto.getRandomValues(new Uint8Array(12))
    const ct = await crypto.subtle.encrypt(
      { name: 'AES-GCM', iv }, this.key,
      new TextEncoder().encode(JSON.stringify(v))
    )
    const out = new Uint8Array(12 + ct.byteLength)
    out.set(iv)
    out.set(new Uint8Array(ct), 12)
    localStorage.setItem(`agora:${k}`, btoa(String.fromCharCode(...out)))
  }

  async get<T>(k: string): Promise<T | null> {
    if (!this.key) throw new Error('Store not initialized')
    const raw = localStorage.getItem(`agora:${k}`)
    if (!raw) return null
    const bytes = Uint8Array.from(atob(raw), c => c.charCodeAt(0))
    const pt = await crypto.subtle.decrypt(
      { name: 'AES-GCM', iv: bytes.slice(0, 12) }, this.key, bytes.slice(12)
    )
    return JSON.parse(new TextDecoder().decode(pt))
  }

  async clear(): Promise<void> {
    Object.keys(localStorage)
      .filter(k => k.startsWith('agora:') && k !== SALT_KEY)
      .forEach(k => localStorage.removeItem(k))
  }
}

export const encryptedStore = new EncryptedStore()
```

### 8.3 Private Messaging (NIP-44 + NIP-59)

```typescript
// src/lib/nostr/dms.ts
// Direct messages use NIP-44 encryption (XChaCha20-Poly1305)
// wrapped in NIP-59 Gift Wrap (hides sender and recipient from relay)

import { nip44, nip59, finalizeEvent, generateSecretKey, getPublicKey } from 'nostr-tools'
import { activePersona } from '$lib/stores/auth'
import { get } from 'svelte/store'

export async function sendPrivateMessage(
  recipientPubkey: string,
  message: string
): Promise<object> {
  const persona = get(activePersona)
  if (!persona) throw new Error('Not logged in')

  const sk = persona.keypair.secretKey
  const pk = persona.keypair.publicKey

  // Inner event: the actual DM (kind:14, NIP-17)
  const dm = finalizeEvent({
    kind: 14,
    created_at: Math.floor(Date.now() / 1000),
    tags: [['p', recipientPubkey]],
    content: nip44.encrypt(message, nip44.getConversationKey(sk, recipientPubkey)),
  }, sk)

  // Outer gift wrap (kind:1059, NIP-59)
  // Uses a random ephemeral key — hides who sent the DM
  const wrapperSk = generateSecretKey()
  const wrapped = finalizeEvent({
    kind: 1059,
    created_at: Math.floor(Date.now() / 1000),
    tags: [['p', recipientPubkey]],
    content: nip44.encrypt(JSON.stringify(dm), nip44.getConversationKey(wrapperSk, recipientPubkey)),
  }, wrapperSk)

  return wrapped
}

export function decryptMessage(
  giftWrap: any,
  ourSecretKey: Uint8Array
): string | null {
  try {
    const ck = nip44.getConversationKey(ourSecretKey, giftWrap.pubkey)
    const inner = JSON.parse(nip44.decrypt(giftWrap.content, ck))
    const ourPk = getPublicKey(ourSecretKey)
    const dmCk = nip44.getConversationKey(ourSecretKey, inner.pubkey === ourPk ? giftWrap.tags.find((t: string[]) => t[0] === 'p')?.[1] : inner.pubkey)
    return nip44.decrypt(inner.content, dmCk)
  } catch {
    return null
  }
}
```

```typescript
export async function panicWipe(): Promise<void> {
  // Overwrite then delete
  Object.keys(localStorage)
    .filter(k => k.startsWith('agora:'))
    .forEach(k => {
      localStorage.setItem(k, 'x'.repeat(64))
      localStorage.removeItem(k)

  await Promise.all(
    dbs.filter(d => d.name?.startsWith('agora'))

         r.onsuccess = () => res()
         r.onerror = () => rej(r.error)
       }))
  )
  sessionStorage.clear()
  window.location.replace('about:blank')
```

## 9. COLLABORATIVE ALGORITHM

### 9.1 Scoring

```typescript
// src/lib/algorithm/scoring.ts
import type { Event as NostrEvent } from 'nostr-tools'
import { VOTE_CONTENT } from '$lib/nostr/kinds'

export interface Weights {
  contentLength:   number
  replyDiversity:  number
  voteQuality:     number
  discussionDepth: number
  timeDecay:       number
}

export const DEFAULT_WEIGHTS: Weights = {
  contentLength:   0.15,
  replyDiversity:  0.25,
  voteQuality:     0.35,
  discussionDepth: 0.15,
  timeDecay:       0.10,
}

export interface ScoreResult {
  total: number
  breakdown: Record<string, number>
  weights: Weights
}

export function scoreEvent(
  event: NostrEvent,
  replies: NostrEvent[],
  reactions: NostrEvent[],
  weights = DEFAULT_WEIGHTS
): ScoreResult {
  const len = event.content.length
  const contentLength =
    len < 50 ? 0.2 : len < 200 ? 0.5 : len < 1000 ? 1.0 : len < 3000 ? 0.8 : 0.6

  const uniqueRepliers = new Set(replies.map(r => r.pubkey)).size
  const replyDiversity = replies.length === 0 ? 0
    : Math.min(1, uniqueRepliers / replies.length)

  const mine = reactions.filter(r => r.tags.some(t => t[0] === 'e' && t[1] === event.id))
  const ins  = mine.filter(r => r.content === VOTE_CONTENT.insightful).length
  const con  = mine.filter(r => r.content === VOTE_CONTENT.constructive).length
  const bri  = mine.filter(r => r.content === VOTE_CONTENT.bridge).length
  const up   = mine.filter(r => r.content === VOTE_CONTENT.upvote).length
  const dn   = mine.filter(r => r.content === VOTE_CONTENT.downvote).length
  const voteQuality = mine.length === 0 ? 0
    : Math.min(1, (ins * 3 + con * 2 + bri * 4 + up) / (mine.length + dn + 1))

  const discussionDepth = Math.min(1, replies.length / 10)

  const ageHours = (Date.now() / 1000 - event.created_at) / 3600
  const timeDecay = 1 / (1 + Math.log1p(ageHours / 6))

  const breakdown = { contentLength, replyDiversity, voteQuality, discussionDepth, timeDecay }
  const total = Object.entries(breakdown).reduce(
    (sum, [k, v]) => sum + (weights[k as keyof Weights] ?? 0) * v, 0
  )

  return { total: Math.round(total * 10_000) / 10_000, breakdown, weights }
}
```

### 9.2 Sort Strategies

```typescript
// src/lib/algorithm/sorting.ts
import type { Event as NostrEvent } from 'nostr-tools'
import { scoreEvent, DEFAULT_WEIGHTS, type Weights } from './scoring'

export type SortStrategy = 'collaborative' | 'newest' | 'oldest' | 'most_discussed' | 'high_signal'

export function sortEvents(
  events: NostrEvent[],
  repliesByEvent: Map<string, NostrEvent[]>,
  reactions: NostrEvent[],
  strategy: SortStrategy,
  communityWeights?: Partial<Weights>
): NostrEvent[] {
  const w = { ...DEFAULT_WEIGHTS, ...communityWeights }

  if (strategy === 'newest') return [...events].sort((a, b) => b.created_at - a.created_at)
  if (strategy === 'oldest') return [...events].sort((a, b) => a.created_at - b.created_at)
  if (strategy === 'most_discussed')
    return [...events].sort((a, b) =>
      (repliesByEvent.get(b.id)?.length ?? 0) - (repliesByEvent.get(a.id)?.length ?? 0)
    )

  const scored = events.map(e => ({
    event: e,
    score: scoreEvent(e, repliesByEvent.get(e.id) ?? [], reactions, w),
  }))

  if (strategy === 'high_signal')
    return scored.sort((a, b) => b.score.breakdown.voteQuality - a.score.breakdown.voteQuality)
                 .map(s => s.event)

  // 'collaborative'
  return scored.sort((a, b) => b.score.total - a.score.total).map(s => s.event)
}
```

```typescript
  try {
    const r = await fetch(
      `${instanceUrl}/api/v3/community?name=${communityName}`,
      { signal: AbortSignal.timeout(8000) }
    )
    if (!r.ok) return null
    const { community_view: cv } = await r.json()
    return {

      name: cv.community.name,
      title: cv.community.title,
      description: cv.community.description ?? '',
      instanceUrl,
      subscribers: cv.counts.subscribers,
      posts: cv.counts.posts,
    }
  } catch { return null }
}

export async function fetchLemmyPosts(
  instanceUrl: string, communityId: number,
  sort = 'Active', page = 1
): Promise<AgoraThread[]> {
  try {
    const r = await fetch(
      `${instanceUrl}/api/v3/post/list?community_id=${communityId}&sort=${sort}&page=${page}&limit=25`,
      { signal: AbortSignal.timeout(8000) }
    )
    if (!r.ok) return []
    const { posts } = await r.json()
    return posts.map((pv: any) => ({
      id: `lemmy:${pv.post.ap_id}`,
      title: pv.post.name,
      body: pv.post.body ?? '',
```

## 11. SEARCH

```typescript
// src/lib/search/index.ts
// Primary: nostr.band API (free, no key, NIP-50 compliant)
// Fallback: client-side filter of cached events

export interface SearchResult {
  events: any[]
  source: 'relay' | 'cache'
}

export async function search(
  query: string,
  options: { kind?: number; communityId?: string; limit?: number } = {}
): Promise<SearchResult> {
  const { kind = 1, limit = 50 } = options

  // Try nostr.band API first (fastest, most comprehensive)
  try {
    const filter: Record<string, unknown> = { kinds: [kind], search: query, limit }
    if (options.communityId) filter['#a'] = [options.communityId]

    const { subscribe } = await import('$lib/nostr/client')
    const events: any[] = []

    await new Promise<void>((resolve) => {
      const timeout = setTimeout(resolve, 5000)
      const sub = subscribe(
        [filter],
        (e) => events.push(e),
        () => { clearTimeout(timeout); resolve() }
      )
    })

    if (events.length > 0) return { events, source: 'relay' }
  } catch { /* fall through */ }

  // Fallback: search local IndexedDB cache
  const { searchCache } = await import('$lib/cache/idb')
  const cached = await searchCache(query, kind)
  return { events: cached, source: 'cache' }
}
```

```typescript
// src/lib/utils/markdown.ts
// External images BLOCKED (would leak IP to image server).
// External links get warning dialog.
// Everything else renders normally.

import { marked } from 'marked'
import DOMPurify from 'dompurify'

const renderer = new marked.Renderer()

renderer.image = (href, _title, text) => {
  // Block external images — render as text link instead
  if (href.startsWith('http') && !href.includes(location.hostname)) {
    return `<span class="blocked-image" title="External image blocked">🖼️ ${text || href}</span>`
  }
  return `<img src="${href}" alt="${text}" loading="lazy" />`
}

renderer.link = (href, _title, text) => {
  const isExternal = href.startsWith('http') && !href.includes(location.hostname)
  if (isExternal) {
    return `<a href="${href}" class="external-link" data-external="true"
              rel="noopener noreferrer">${text}</a>`
  }
  return `<a href="${href}">${text}</a>`
}

export function renderMarkdown(md: string): string {
  const html = marked(md, { renderer, breaks: true })
  return DOMPurify.sanitize(html as string, {
    ALLOWED_TAGS: ['p','br','strong','em','code','pre','blockquote',
                   'ul','ol','li','h2','h3','h4','a','img','span','hr'],
    ALLOWED_ATTR: ['href','src','alt','class','title','data-external'],
    ALLOW_DATA_ATTR: false,
  })
}
```

## 17. CODE GENERATION RULES

```
1. COMPLETE FILES
   Every generated file is complete and immediately runnable.
   No TODOs, no placeholder functions, no "implement this later".

2. MOBILE FIRST
   Every component works on a 375px screen.
   Touch targets ≥ 44px. No hover-only interactions.

3. ACCESSIBILITY
   ARIA attributes on first write, not as a polish pass.
   Every interactive element is keyboard accessible.
   All images have alt text. Live regions for dynamic content.

4. HONEST ERROR HANDLING
   Network failures show clear, actionable messages.
   "Relay connection failed — check your internet connection"
   not "Error: WebSocket closed with code 1006".

5. PROGRESSIVE ENHANCEMENT
   Core reading works without JavaScript (SSR).
   Posting requires JS. Document this.

6. NO EXTERNAL REQUESTS FROM app.html
   Zero CDN links. Zero external scripts. Zero Google Fonts.
   System font stack. All assets in build output.
   CSP header enforces this.

7. SECURITY
   No .unwrap()-equivalent (no unchecked .json()!).
   Every crypto operation has error handling.
   User-facing crypto errors explain what happened
   and what to do, not stack traces.

8. WHEN UNCERTAIN
   Choose: simpler, more private, more honest.
   Document the choice in a comment.
```
